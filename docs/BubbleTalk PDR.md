# PRD：BubbleTalk对话原生 AI 教育软件

## 1. 背景与机会

传统学习产品的两大痛点：

1. **启动成本高**：打开课程像打开一座矿山，用户需要规划、投入长时段注意力。
2. **学习过程缺乏互动与个性化推进**：多数产品要么内容灌输，要么题海训练，缺少“像聊天一样的探索”。

大模型带来新的可能：把“学习”重构为**以对话为主轴**的交互过程，结合自适应测评、故事化讲解、多模态材料与跨主题联动，让用户获得：

- 低启动成本
- 高参与感与自主感
- 可验证的掌握度与迁移能力

本产品定位：**对话是一等公民（Conversation First）的 AI 原生学习软件**。

------

## 2. 产品目标

### 2.1 北极星指标（North Star）

**每次学习会话（6–12 分钟）带来“可迁移理解”增益**
衡量方式：会话结束迁移题通过率 + 24h 复习命中率提升。

### 2.2 核心目标

- G1：把学习启动成本降到“点一下泡泡即可开始”
- G2：对话推进不是陪聊，必须有“结构化认知推进”
- G3：跨领域通用：经济学、编程、历史、人文社科、职场技能等均可承载
- G4：个性化泡泡：结合用户背景生成“更想点”的入口
- G5：安全与可信：明确标注不确定性，避免胡编与越界建议

### 2.3 非目标（V1 不做）

- NG1：完整替代系统课程体系（V1 以微型学习单元为主）
- NG2：严肃证书考试闭环（可后续扩展）
- NG3：社交社区、排行榜等强社交机制（先验证学习增益）

------

## 3. 用户画像与场景

### 3.1 目标用户

- U1：碎片时间学习者（每天 5–15 分钟）
- U2：兴趣驱动探索者（广泛好奇，但需要结构化引导）
- U3：实用主义提升者（希望把知识用于解释现象/做决策/工作解决问题）
- U4：深度学习者（愿意更长对话与更难迁移题）

### 3.2 核心使用场景

- S1：通勤/睡前打开 app，想“学点有用的”
- S2：遇到现实问题，想快速建立概念框架（例如“为什么房租涨”“为什么缓存抖动”）
- S3：想把已学知识串联成体系，形成长期记忆与迁移能力
- S4：希望通过对话提出自己的问题，而不是被动听课

## 4. 产品概念：泡泡不是“概念”，而是“可进入的学习入口”

### 4.1 泡泡定义（Bubble）

泡泡代表一个**学习入口（Learning Entry）**，它可由三种“壳”承载：

- **概念入口**：某个杠杆概念（例如“机会成本”）
- **情境入口**：贴近用户生活/工作的问题（例如“我该接这个 Offer 吗？”）
- **事件入口**：时事/历史/案例引发的解释需求（例如“为什么价格战会两败俱伤？”）

V1 推荐以“情境入口”为主，概念入口为辅。用户更愿意点情境，系统再把它映射到概念骨架。

### 4.2 泡泡生成策略（个性化）

泡泡标题与描述应结合用户背景，生成“更想点”的表达：

- 经济学：从“机会成本”变成“你周末加班到底值不值？”
- 编程：从“CAP 定理”变成“为什么你的服务高峰期必炸？”
- 心理学：从“延迟满足”变成“为什么你总在深夜报复性刷手机？”
- 职场：从“激励相容”变成“为什么好制度反而让人摆烂？”

每个泡泡要能回溯到一个或多个 Concept ID，保证学习结构可追踪。

------

## 5. 信息架构与主流程

### 5.1 首页（Bubble Home）

- 泡泡云（大小代表“推荐强度”）
- 顶部可切换“目标模式”：
    - 探索（广度）
    - 精进（深度）
    - 复习（巩固）
    - 解决问题（情境驱动）
- 入口：搜索（按问题/概念/领域）、收藏、历史会话

### 5.2 进入泡泡后流程（Session Flow）

1. **快速自适应测评（2–4 题）**
2. **对话研讨课（Conversation Seminar）**
    - 多角色音频讲述 + 用户随时插话
    - 强制的“用户输出动作”嵌入（复述/判断/举例/迁移）
    - 允许分叉问题进入“待回收栈”
3. **退出机制**
    - 用户主动退出 或 对话自然结束
    - 离场仪式（Exit Ticket）：1 道迁移题 + 1 句自我解释
4. **会话总结与更新**
    - 掌握度、误解标签、兴趣偏好、下一步建议泡泡

------

## 6. 关键模块设计

### 6.1 自适应测评模块（Adaptive Quiz）

**目标**：定位起点、识别典型误解、为对话定锚。

题型建议（V1）：

- 直觉判断题（低压、快）
- 干扰项映射误解（每个选项对应 Misconception Tag）
- 轻迁移题（换场景但同结构）

更新规则（V1 简化版）：

- Mastery（掌握度 0–1）
- Misconception Tags（误解标签集合）
- Confidence（置信度：基于对题耗时、解释质量、稳定性）

### 6.2 对话引擎：导演状态机（Director State Machine）

对话自由插话但主线不散的核心：一个**不出声的导演**维护学习目标与路由。

导演维护的状态：

- CurrentObjective：本节核心关系（只允许 1 个主目标）
- SubGoals：澄清点、边界点、迁移点
- QuestionStack：用户分叉问题栈（Branch）
- UserModelSnapshot：当前会话内的掌握与误解

导演对用户插话分类：

- Clarify（澄清当前内容）立刻答
- Deepen（加深当前内容）立刻答并回主线
- Branch（开新话题）入栈，主线结束后回收
- Meta（节奏/难度/偏好）调整策略

对话阶段（固定 5 段）：

1. Hook（故事/现象引入）
2. Model（核心关系建模，短讲）
3. Check（用户复述或判断，必须输出）
4. Challenge（反例与边界条件）
5. Transfer（新场景迁移应用）

**硬约束**：每 90 秒必须触发一次“用户输出动作”。

### 6.3 多角色系统（Role System）

角色不固定教授/同学，角色是可配置模板，用来达成不同学习体验。

角色模板（示例）：

- 专家型：教授、经济学家、资深工程师
- 同伴型：同学、同事、学习搭子
- 主持型：访谈主持人，负责提问与节奏控制
- 挑战型：辩手/审稿人，专门提出反例与漏洞
- 故事型：讲故事的记者/纪录片旁白，负责案例叙事
- 教练型：只提问不讲解，苏格拉底式引导

角色选择策略：

- 根据领域默认组合（经济学：专家+主持；编程：同伴+挑战；历史：记者+专家）
- 根据用户偏好可切换（“更严肃”“更轻松”“更多例子”“更快节奏”）
- 会话中允许“一键换阵容”

音频呈现：

- V1：TTS 语音输出（区分角色音色）
- 可选：静音模式（纯文字对话）

### 6.4 泡泡呈现形式（Bubble Card UI）

泡泡本体展示建议统一为“可点的卡片泡泡”，支持多模态封面：

- 纯文字泡泡：标题 + 一句钩子（默认）
- 生成插画泡泡：简洁漫画风封面（增强吸引力）
- 轻动效泡泡：微动效，不走短视频路线
- 音频预告泡泡：3–5 秒角色开场一句话（可选）

V1 推荐：**文字为主 + 可选生成插画封面**
原因：生成视频成本高且容易滑向娱乐化。

### 6.5 内容与知识结构（Concept Graph）

为跨领域扩展与稳定学习效果，必须把内容结构化：

- Concept（概念节点）：concept_id、定义、核心关系、前置概念、典型误解、适用边界
- Scenario（情境节点）：scenario_id、场景描述、关联概念、常见错误推理
- Asset（素材）：故事案例、图解、漫画、练习题模板、迁移题模板

泡泡（Entry）映射到：Scenario → Concept（一个主概念 + 若干辅助概念）

## 7. 个性化与推荐

### 7.1 用户建模（User Model）

三层记忆与状态：

- Session Memory（会话短期）：当前目标、问过的问题、当前误解
- User Memory（个人长期）：兴趣分布、掌握度矩阵、误解画像、偏好节奏与风格
- External Knowledge（外部知识）：书籍/课程/文章等，经用户授权接入

用户输入来源：

- 显式：用户选择领域、目标、偏好；可选导入数据（需授权）
- 隐式：泡泡点击、停留、提问类型、退出时机、复习命中率

### 7.2 推荐目标函数（避免娱乐化）

泡泡推荐分数建议多目标：

- InterestScore（兴趣）
- LearningNeedScore（欠账与误解）
- ForgettingRiskScore（遗忘风险）
- DiversityScore（多样性）
- FatigueScore（疲劳抑制）

推荐策略：

- 70% 满足兴趣与目标
- 20% 修复欠账与误解
- 10% 探索与拓展（轻推新领域）

### 7.3 画像推断原则（隐私友好）

从对话推断用户偏好时，采取“候选假设 + 轻确认”：

- “我猜你更偏好案例驱动学习，对吗？是/否”
  避免系统暗自建立敏感画像。

------

## 8. 反馈与学习数据更新

会话结束更新：

- Mastery（概念掌握度）
- Misconception Tags（误解画像）
- TransferScore（迁移能力）
- Engagement（参与度：不是时长，重点是有效输出）
- Next Best Bubble（下一步推荐）

离场总结输出（克制三行）：

1. 你学到的核心关系
2. 你最可能的误解
3. 下一步建议（一个泡泡 + 理由）

------

## 9. 安全、合规与边界

- 明确免责声明：教育解释不等于个体化决策建议
- 对金融/医疗/法律等高风险领域：
    - 提供一般性科普
    - 避免具体投资建议与个股指令
    - 必要时提示咨询专业人士
- 内容可信：
    - 对不确定内容标注不确定性
    - 对外部知识引用给出处（可在会话末附“来源卡片”）

## 10. MVP 范围（V1）

### 10.1 领域范围

- V1 做 2 个领域以验证跨域能力：
    - 经济学（示例）
    - 编程/分布式系统（示例）
      每个领域 10–15 个高杠杆泡泡入口（以情境为主）。

### 10.2 功能范围

必须有：

- 泡泡首页（推荐+搜索+历史）
- 泡泡进入后的自适应测评（2–4 题）
- 对话研讨课（导演状态机 + 多角色）
- Exit Ticket（迁移题 + 一句话解释）
- 用户模型更新与复习入口（24h 复习泡泡）

可选（加分项）：

- 泡泡封面生成插画
- 角色音色选择与语速控制
- “分叉问题栈”的可视化回收列表

不做：

- 用户社交关系链
- 完整课程体系与章节目录
- 大规模 UGC 内容市场（先保证质量）

## 11. 成功指标与实验

### 11.1 核心指标

- Learning Gain：Pre vs Post 迁移题提升
- 24h Retention：次日复习命中率
- Misconception Clearance：典型误解消除率
- Effective Output Rate：用户有效输出占比（复述/解释/迁移尝试）

### 11.2 辅助指标

- Session Completion Rate（完成率）
- Return Rate（次日回访）
- Bubble CTR（泡泡点击率）
- Satisfaction（主观满意度）

### 11.3 A/B 实验方向（V1 重点）

- 角色组合：专家+主持 vs 专家+同伴
- 插话策略：即时打断 vs 记录后回收
- 讲解比例：短讲 20s vs 40s
- Exit Ticket：仅迁移题 vs 迁移题+一句话解释

------

## 12. 风险与对策（红队）

1. 对话漂移变陪聊

- 对策：导演状态机、阶段硬约束、用户输出动作频率

1. 用户只听不说，产生理解幻觉

- 对策：强制 Check 阶段，短讲后必须输出

1. 推荐变成兴趣流，学习债务不还

- 对策：多目标推荐函数，加入遗忘风险与欠账权重

1. 生成内容可信度问题

- 对策：领域素材库+结构化概念卡，生成遵循骨架，必要时引用来源卡

1. 个性化推断越界引发反感

- 对策：候选假设+轻确认；透明可编辑的个人偏好面板

------

## 13. 里程碑与迭代路线

- M0：概念图谱与泡泡数据结构定稿（Concept/Scenario/Asset/Entry）
- M1：经济学领域 10 个泡泡可用，完成“测评→对话→退出→更新”闭环
- M2：加入编程领域 10 个泡泡，验证跨域通用性
- M3：24h 复习系统上线（遗忘风险驱动的复习泡泡）
- M4：泡泡封面插画生成与角色音色扩展
- M5：UGC 轻量化（用户提交场景问题，系统生成泡泡入口，审核后入库）

## 14. 附录：示例泡泡（经济学领域）

（注意：展示给用户的是情境标题，系统内部映射 concept_id）

- “周末加班值不值？” → 机会成本 + 时间偏好
- “为什么打折越买越亏？” → 边际效用 + 沉没成本
- “价格战为什么停不下来？” → 囚徒困境 + 承诺
- “保险为什么让人更敢冒险？” → 道德风险
- “二手市场为什么更容易被骗？” → 逆向选择

- 经济学：经济学家（专家）+ 播客主持人（主持）+ 反例挑战者（挑战）
