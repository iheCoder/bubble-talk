# 机会成本 - 剧本

## 剧本ID
script_opportunity_cost

## Entry ID
opportunity_cost

## 原始故事（Original Story）

### 主题
通过对话式节目，让用户深入理解"机会成本"的核心概念——不是看你花了多少钱，而是看你放弃了什么。

### 核心冲突
大多数人的直觉是：赚钱就是赚了，不花钱就是省了。但经济学告诉我们：你选择A，就意味着放弃B的价值，这才是真正的成本。

### 开场钩子（ColdOpen - 2分钟）
**场景**：周末加班 vs 演唱会

主持人抛出一个生活场景：
"周末你原本约了朋友去看演唱会，票价500块。但公司临时喊你加班，给你800块加班费，你去了。后来朋友发照片给你看，演唱会超精彩。

你觉得这次加班，你是赚了还是亏了？"

**预期用户反应**：
- 大部分人会说"当然是赚了，800-500=300"
- 少数人可能会犹豫"好像有点亏，演唱会很精彩"

**冲突点**：用户的直觉（赚了300）vs 经济学思维（失去了演唱会体验+800的机会成本）

### 推进1：定义澄清（DeepDive - 3分钟）

经济学家介入：
"这是典型的直觉思维。但我想问你一个问题：如果你去了演唱会，那个周末你失去了什么？"

停顿，让用户思考。

"你失去的不是500块票钱，而是800块加班费。所以机会成本是800，不是500。"

"换句话说，你这次选择加班，花掉的不是时间或500块票钱，而是「演唱会体验 + 失去的800块收入」。如果演唱会对你的价值（快乐、回忆、社交）超过800，那你其实是亏了。"

**关键概念**：机会成本 = 你放弃的最优选择的价值

### 推进2：迁移练习（Montage - 2分钟）

主持人抛出新例子检验理解：
"那我再问你一个：假如你下班后有两个选择——
- 选项A：去健身房锻炼（会员卡已经买了）
- 选项B：在家追剧（不花钱）

如果你选择追剧，机会成本是什么？"

**预期用户反应**：
- 正确：机会成本是放弃的健身收益（健康、精力）
- 错误：机会成本是0，因为追剧不花钱

**澄清**：会员卡的钱已经是沉没成本，不影响这次选择的机会成本。机会成本只看「未来」你放弃了什么。

### 推进3：边界问题（Debate - 2分钟）

经济学家抛出边界问题：
"但这里有个陷阱很多人会踩：是不是所有被放弃的选项都算机会成本？"

"比如你选了追剧，放弃了健身、放弃了学习、放弃了社交、放弃了做家务...这么多选项，哪个才是机会成本？"

停顿，等用户思考。

**答案**：机会成本是「在所有被放弃的选项中，价值最高的那一个」。

**为什么重要**：这澄清了"最优选择"的概念，避免用户认为所有放弃的都算成本。

### 收束（Wrap - 2分钟）

主持人总结：
"完美！机会成本就是——在所有被放弃的选项中，价值最高的那一个。"

"回到最开始的例子：你加班赚800，放弃了演唱会。如果演唱会对你的价值（快乐、回忆、社交）超过800，那你这次就是亏了。"

"这就是为什么经济学家总说：免费的往往最贵——因为机会成本可能很高。"

"你学会了一个思考决策的新角度。下次做选择时，不妨问问自己：我放弃了什么？那才是真正的成本。"

### 测评（ExitTicket - 1分钟）

主持人：
"最后一个问题，检验你是否真的理解了：

你在超市看到两种牛奶：
- A品牌：10块，口感一般
- B品牌：15块，超好喝

你选了A，省了5块。请问你的机会成本是什么？"

**正确答案**：机会成本是B品牌牛奶带来的「超好喝」体验，不是5块钱。

**评分标准**：
- 答对：完全理解机会成本是放弃的价值，不是省下的钱
- 答"5块钱"：仍然停留在显性成本思维
- 答"B品牌"但没提价值：部分理解

## 角色设定

### 主持人（host）
- **职责**：控节奏、抛冲突、引导对话、制造代入感
- **语言风格**：轻松幽默、生活化、善于用场景化例子
- **关键时刻**：开场钩子、迁移练习、总结收束

### 经济学家（economist）
- **职责**：解释机制、澄清概念、给出框架
- **语言风格**：严谨但不端着、善于用反问引导思考
- **关键时刻**：定义澄清、边界问题

### 怀疑者（skeptic）- 可选
- **职责**：代表普通人直觉、提出反例、制造张力
- **语言风格**：接地气、敢于挑战、说出用户心声
- **关键时刻**：在用户犹豫时代言"但是免费的不是更好吗？"

## 对齐策略

### FOLLOW模式（对齐度 > 0.7）
- 严格按照剧本的场景顺序推进
- 用户的反应符合预期时，自然过渡到下一段
- 保持剧本设定的节奏和张力

### ADAPT模式（0.4 < 对齐度 ≤ 0.7）
- 用户提前理解了某个概念 → 跳过对应的澄清环节
- 用户提出了剧本外的好问题 → 临时插入回应，然后回到主线
- 用户卡在某个点 → 增加一个中间例子

### REWRITE模式（对齐度 ≤ 0.4）
- 用户完全偏离主题 → 用HookBack拉回
- 用户疲惫或困惑 → 降低难度，用更简单的例子
- 用户已经很理解 → 直接跳到边界问题或测评

## 关键原则

1. **故事优先**：像电影对话，不是PPT讲课
2. **冲突驱动**：每个片段都有一个小冲突需要解决
3. **用户中心**：观察用户反应，动态调整
4. **电影感**：节奏紧凑，不拖沓，有起承转合
